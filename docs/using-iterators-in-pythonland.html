Chapter 24 Using iterators in PythonLand

24.1 枯燥但必須知道的一些理論

24.1.1 定義：Iterable, Iterator

  • 先來定義這兩個名詞
  • Iterable (可迭代物件):
    • 直觀定義：可以對他用for迴圈的object，我們叫他iterable。所以iterable指是一種特殊的object，哪種object？可對他做for迴圈的那種object。例如list, strings, dictionary都是iterable型的object
    • 正式的定義是：可以對他做iter()的object，叫iterable
  • Iterator (迭代器):
    • 剛剛說，可以對他做iter()的object，我們叫他iterable，所以，做完後的object，我們叫他iterator
    • 例如： a = [1,2,3]， a是 iterable(因為，我可以對他做iter()); 而aa = iter([1,2,3])，aa是iterator
    • iterator的特性是，你對他用一次next()，他就會吐出一個值。你再next()一次，他再吐一個值。如果此iterator有n個element，你要next n次，才會全部吐完。
    • 所以確認一個物件是不是iterator，你就next()他看看就知道了。

24.1.2 練習分辨iterable和iterator

flash1 = [1,2,3,4,5]
flash2 = iter(flash1)
  • 我們發現對flash1做iter()是ok的，所以flash1是可以iter的，我們就知道falsh1是iterable型的object
  • 而此時，flash2是被iter()完的物件，我們就稱它為iterator
  • 去print()看看兩者，可以看到iterable會直接給你看結果，iterator則是很矜持，不給你看，只有你對他用next()時才會給你看
print(flash1)
#> [1, 2, 3, 4, 5]
print(flash2)
#> <list_iterator object at 0x13f54f9a0>
  • 對flash1和flash2用next()，可以看到iterable會回你error，因為只有iterator才對next()有反應。對iterator用next()，他則會一次吐一個值給你
next(flash1)
#> Error in py_call_impl(callable, dots$args, dots$keywords): TypeError: 'list' object is not an iterator
#> 
#> Detailed traceback: 
#>   File "<string>", line 1, in <module>
next(flash2)
#> 1
next(flash2)
#> 2
next(flash2)
#> 3
next(flash2)
#> 4
next(flash2)
#> 5
  • 來個反例吧：
num = 12345
num_iter = iter(num)
#> Error in py_call_impl(callable, dots$args, dots$keywords): TypeError: 'int' object is not iterable
#> 
#> Detailed traceback: 
#>   File "<string>", line 1, in <module>
  • 看到了吧，不是隨便一個object都可以對他做iter()，error message直接跟你說，int這種object不是iterable的object

24.1.3 用range()做出來的object，是iterable

aa = range(0,5)
aaa = iter(aa)
  • 我們對aa做iter()，發現是ok的，所以aa是iterator
  • 其實我們早就知道了，因為從直觀定義來看，range()物件是可以用for迴圈的，所以他是iterator
  • 但有意思的是，既然range()是iterable，那我做完iter()的aaa就是iterator，那我每對他做一次next()，他就應該會吐一個值來：
next(aaa)
#> 0
next(aaa)
#> 1
next(aaa)
#> 2
next(aaa)
#> 3
next(aaa)
#> 4

24.1.4 傲嬌的iterable，可用list()或加星號來揭開面紗

  • 剛剛知道range()的object是iterable，當你對他用print()的時候，你會發現他不給你看裡面啊！！
aa = range(0,5)
print(aa)
#> range(0, 5)
  • 所以，你要把他轉成list，他才會乖乖給你看
print(list(aa))
#> [0, 1, 2, 3, 4]
  • 或是，加星號：
print(*aa)
#> 0 1 2 3 4
  • 喔嗚…，加星號所print出的東西，看起來是一個一個print出來的，怎麼跟next()做出來的效果這麼像？that’s right，其實加星號的作用就是：
    • 先做iter()把他轉成iterator
    • 再一路next()到底
aaa = iter(aa)
for i in aa:
  print(next(aaa))
#> 0
#> 1
#> 2
#> 3
#> 4

24.1.5 我到底為什麼要學iterators…

  • 這邊很快的來講一個應用。在大數據的時代，我們常常從DB拉下來的data.frame就非常巨大，所以通常不會直接全部拉下來然後存成一個變數，因為這樣會讓你的記憶體直接炸裂
  • 那，取而代之的做法，就是先拉一批進來，然後做些處理，得到結果後，刪掉這批資料，再拉下一批進來。等到最後一批進來並得到結果後，再把剛剛前面所有結果整合起來，得到最終結果
  • 那，這邊的一批一批依序進來，不就是iterator在做的事嗎？
  • 所以現在給一個實際情境，假設我們電腦硬碟中，有一個超大的excel表，裡面是放twitter資料。然後，這筆資料中有一個欄位是lang(語言)。我們想做的，是去得出這一欄的次數分配表
  • 那我就可以這樣做：
    • 寫回圈，每次讀進1000行的table進來，統計出次數分配表，然後刪掉這個table
    • 依序讀入並更新，直到最後一批資料進來
  • 直接來看code吧：
import pandas as pd
# Initialize an empty dictionary: counts_dict
counts_dict = {}

# Iterate over the file chunk by chunk
for chunk in pd.read_csv("./data/tweets.csv", chunksize=10):

    # Iterate over the column in DataFrame
    for entry in chunk['lang']:
        if entry in counts_dict.keys():
            counts_dict[entry] += 1
        else:
            counts_dict[entry] = 1

# Print the populated dictionary
print(counts_dict)
#> {'en': 97, 'et': 1, 'und': 2}
  • nice，我們更進一步，寫一個function來做：
# Define count_entries()
def count_entries(csv_file, c_size, colname):
    """Return a dictionary with counts of
    occurrences as value for each key."""
    
    # Initialize an empty dictionary: counts_dict
    counts_dict = {}

    # Iterate over the file chunk by chunk
    for chunk in pd.read_csv(csv_file, chunksize=c_size):

        # Iterate over the column in DataFrame
        for entry in chunk[colname]:
            if entry in counts_dict.keys():
                counts_dict[entry] += 1
            else:
                counts_dict[entry] = 1

    # Return counts_dict
    return counts_dict
# Call count_entries(): result_counts
result_counts = count_entries('./data/tweets.csv',10,'lang')

# Print result_counts
print(result_counts)
#> {'en': 97, 'et': 1, 'und': 2}

24.2 playing with iterators

  • 我們之前學過，對iterable可以使用for迴圈，那，從現在開始，要多教兩個可以對iterable使用的大招：enumerate()和zip()

24.2.1 enumerate

  • enumerate()是一個function，從字義來看，e-就是“使”的意思，後面的“numerate”就是number的動詞，所以整個就是：給number！
    所以可以想像，如果我對一個iterable(例如，list)做enumerate，那他會保留原本的list，但同時，這個list的每一個element，都被標記上所屬的number！
  • 更精確的說， enumerate(iterable_obj)，可以將iterable object轉成enumerate object，此object會是一組tuple的序列，每個tuple都是index-value pair。看一下例子：
mutants = ['charles xavier', 
            'bobby drake', 
            'kurt wagner', 
            'max eisenhardt', 
            'kitty pryde']
enum_mutants = enumerate(mutants)
print(enum_mutants)
#> <enumerate object at 0x13f6b5a40>
  • 對mutant這個iterable做完enumerate後，他現在是一個enumerate object。但他很傲嬌的不給你看裡面有什麼。所以，用老招，把他改成list，看看裡面有什麼：
print(list(enum_mutants))
#> [(0, 'charles xavier'), (1, 'bobby drake'), (2, 'kurt wagner'), (3, 'max eisenhardt'), (4, 'kitty pryde')]

  • 果然是個index-value的pair，“charles xavier”現在就和他所屬的index=0綁在一起

  • 那…，做enumerate有啥好處？看到tuple，就想到他可以被輕易拆分並assign到兩個variable，那就可以增加for迴圈的靈活性，例如：

for index1, value1 in enumerate(mutants): 
    print(index1, value1)
#> 0 charles xavier
#> 1 bobby drake
#> 2 kurt wagner
#> 3 max eisenhardt
#> 4 kitty pryde
  • enumerate也可以指定index要從幾開始編，例如：
for index2, value2 in enumerate(mutants, start=1): #改從index=1開始編號
    print(index2, value2)
#> 1 charles xavier
#> 2 bobby drake
#> 3 kurt wagner
#> 4 max eisenhardt
#> 5 kitty pryde
  • 最後，回到剛剛傲嬌的討論上，我們在iterator那邊有討論到，iterator都很傲嬌，你要用next()他才會給你看裡面，且一次看一個。那…剛剛發現enumerate object那麼傲嬌，他會不會是iterator？
  • sure…他就是iterator，所以現在學到了，對一個iterable做iter()，他會變iterator，對一個iterable做enumerate()，他也會變成iterator：
enum_mutants = enumerate(mutants)
next(enum_mutants)
#> (0, 'charles xavier')
next(enum_mutants)
#> (1, 'bobby drake')
next(enum_mutants)
#> (2, 'kurt wagner')
next(enum_mutants)
#> (3, 'max eisenhardt')
next(enum_mutants)
#> (4, 'kitty pryde')

24.2.2 zip

  • 講到zip就想到壓縮時的打包，而壓縮時，就是要放一堆東西進去，所以…zip就字面上的意思，就是要幫你打包一堆iterable!!
  • 所以zip的用法，是放進任意數量的iterable進去，例如以下這樣：
#先列出三個list，每個list都有五個element
mutant  = ['charles xavier', 'bobby drake'  , 'kurt wagner'  , 'max eisenhardt', 'kitty pryde']
aliases = ['prof x'        , 'iceman'       , 'nightcrawler' , 'magneto'       , 'shadowcat']
powers  = ['telepathy'     , 'thermokinesis', 'teleportation', 'magnetokinesis', 'intangibility']

mutant_zip = zip(mutants, aliases, powers)
  • ok，我打包好了，他現在變成一個zip object
print(mutant_zip)
#> <zip object at 0x13eae3340>
  • 很傲嬌，不給看，那一樣用老招，把他轉成list後print出來
print(list(mutant_zip))
#> [('charles xavier', 'prof x', 'telepathy'), ('bobby drake', 'iceman', 'thermokinesis'), ('kurt wagner', 'nightcrawler', 'teleportation'), ('max eisenhardt', 'magneto', 'magnetokinesis'), ('kitty pryde', 'shadowcat', 'intangibility')]
  • 發現什麼？發現zip在做的，就是把大家的element i 湊成一個tuple， for i 從第一個element到最後一個element
  • 同時，也知道對一個iterable做zip()後，他也變成iterator，所以可用next()把他叫出來
mutant_zip = zip(mutants, aliases, powers)
print(next(mutant_zip))
#> ('charles xavier', 'prof x', 'telepathy')
print(next(mutant_zip))
#> ('bobby drake', 'iceman', 'thermokinesis')
print(next(mutant_zip))
#> ('kurt wagner', 'nightcrawler', 'teleportation')
print(next(mutant_zip))
#> ('max eisenhardt', 'magneto', 'magnetokinesis')
print(next(mutant_zip))
#> ('kitty pryde', 'shadowcat', 'intangibility')
  • 那…這樣做有啥好處？一樣咩，他是tuple，所以可以任意拆分並assign給任意variable，所以剛剛enumerate()是幫你吐原來的element + 他所屬的index，那現在zip()就等於可以同時幫你吐任意個你輸進去的iterable:
for value1, value2, value3 in zip(mutants, aliases, powers):
    print(value1, value2, value3)
#> charles xavier prof x telepathy
#> bobby drake iceman thermokinesis
#> kurt wagner nightcrawler teleportation
#> max eisenhardt magneto magnetokinesis
#> kitty pryde shadowcat intangibility
